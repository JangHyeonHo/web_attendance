# Phase 6 — 관리자 세분화(인사관리자) + 휴가 관리

요구 3(관리자 세분화)·4(휴가). 3을 먼저 구현, 4는 아래 법적 기준을 사용자 검증 후 착수.

## A. 관리자 세분화 (요구 3 — 이번 구현)

### 배경
회사 직권을 총관리자 1인에 집중하면 리스크 → **인사관리자(HR_ADMIN)** 분리. 회사 내 계층 3단계:
`MEMBER < HR_ADMIN(인사관리자) < TENANT_ADMIN(총관리자)`. (서열 비교는 두지 않음 — 화이트리스트 유지)

### 권한 매트릭스

| 기능 | MEMBER | HR_ADMIN(인사) | TENANT_ADMIN(총) |
|------|:---:|:---:|:---:|
| 본인 출결/정정(W005·W006) | ✓ | ✓ | ✓ |
| 멤버 목록·등록·스케줄·상태·삭제·초대재발송(W009) | | ✓ | ✓ |
| 공휴일 관리(W013) | | ✓ | ✓ |
| **역할 지정**(멤버↔인사↔총, `PUT /members/{id}/role`) | | | ✓ |
| 회사 메일 템플릿(W014) | | | ✓ |
| (후속) 휴가 부여·승인 | | ✓ | ✓ |

- 근거: **인사관리자는 사람·근태·(후속)휴가**를 다루되, **관리자 임명과 회사 대외 설정(메일)은 총관리자**만 — 직권 분산.
- 역할 지정만 총관리자 전용이므로 경로를 세분: `/tenant/members/*/role`·`/tenant/mail-templates/**` = 총관리자, 그 외 `/tenant/**` = 인사+총.
- **마지막 총관리자 보호 유지**: 총관리자 0명이 되면 아무도 임명 불가 → 강등/비활성 409. 인사관리자 수는 무제한·보호 없음.

### 구현 지점
- `Role`에 `HR_ADMIN` 추가(name 직렬화 — 순서 무관, DB VARCHAR).
- `RoleInterceptor` 규칙(선언 순서 = 첫 매칭): `/tenant/members/*/role`·`/tenant/mail-templates/**`(총만) → `/tenant/**`(인사+총) → `/attendance/**`(멤버+인사+총).
- `Screen`: W005·W006·W009·W013에 HR_ADMIN 추가, W014는 총만.
- `NavigationService.homeOf(HR_ADMIN)=ATTENDANCE`.
- `MemberRoleRequest`: TENANT_ADMIN|HR_ADMIN|MEMBER 허용(SYSTEM_ADMIN 400).
- 프론트: 헤더 메뉴 role 분기(인사=멤버·공휴일, 총=+메일템플릿), 멤버 화면 역할 배지·승격/강등에 인사관리자 추가(역할 지정 UI는 총관리자에게만).

## B. 휴가 관리 (요구 4 — 법적 기준 검증 후 착수) ⚠ 사용자 검증 필요

> 자동 딥리서치가 법령 사이트 fetch 전부 차단으로 무산 → 아래는 모델 지식 기반 **초안**. 시행 전 검증 요망.

### B-1. 법정 최소 기준 요약(한/일 대비)

| 항목 | 한국(근로기준법 §60·61) | 일본(労基法 §39) |
|------|------|------|
| 발생 요건 | 1년간 80% 이상 출근 → **15일** | 6개월 계속근무 + 8할 출근 → **10일** |
| 초기(1년 미만) | 1개월 개근 시 1일, **최대 11일**(월 단위) | (6개월까지 없음) |
| 근속 가산 | 3년↑ 매 2년 1일, **한도 25일** | 근속표: 1.5y 11 / 2.5y 12 / 3.5y 14 / 4.5y 16 / 5.5y 18 / **6.5y↑ 20일** |
| 파트타이머 | (소정근로 비례 판례·행정해석) | **比例付与**(주 소정일 4일↓ & 30h↓) |
| 시간단위 | 명문 규정 없음, 취업규칙/합의로 반차·시간연차 실무 | **時間単位年休** 노사협정 시 **연 5일** 한도 |
| 사용촉진/의무 | **연차사용촉진**(§61) 시 미사용 수당 면제 | 연 10일↑ 부여자 **연 5일 시기지정 취득 의무** |
| 시효/이월 | 사용기간 **1년**(이월 원칙 없음), 미사용→수당(수당청구 3년) | **2년**(이월 가능 — 최대 2년분 보유) |

### B-2. 법정 vs 회사 재량

- **법정(필수)**: 연차(위), 산전후·육아휴직(별도 법), 생리휴가(KR §73 무급 가능 / JP §68), 공민권 행사(KR §10 / JP §7).
- **회사 재량(취업규칙)**: 경조사·병가(유급 여부 회사 결정)·여름/리프레시 휴가 등.

### B-3. 데이터 모델 시사점(합의: 법정 자동 계산 + 수동 조정 / 일 단위 + 반일·시간 허용)

- `leave_type`(테넌트별 커스텀): 코드·명칭·유급여부·부여단위(DAY/HOUR)·승인필요·연차성격(ANNUAL 여부). **연차는 기본 시드 종류**로 법정 자동 계산, 그 외 회사 정의.
- `leave_grant`(부여): userId·typeId·**분 단위 수량**(1일=소정근로시간, 예 480분)·유효시작·**만료일**(KR 1년/JP 2년)·출처(AUTO 법정/MANUAL 조정)·메모. 잔여 = Σ부여 − Σ승인사용.
- `leave_request`(신청): userId·typeId·기간(일 또는 시간대)·**차감분**·사유·상태(PENDING/APPROVED/REJECTED/CANCELED)·승인자. 승인 시에만 잔여 차감.
- **단위**: 잔여·차감은 내부 **분(minutes)**로 관리, 표시는 '일(day) + 반차/시간'. 1일=개인 소정근로시간(work_schedule 기반)으로 환산.
- **워크플로**: 인사/총관리자 부여(또는 법정 자동) → 멤버가 W0xx 휴가 신청(종류·날짜/시간·사유) → 관리자 승인 시 차감·확정. 승인 휴가일은 출결 달력(W006)에 표시.

### B-4. 검증 요청 포인트(착수 전)
1. 연차 자동 계산의 정확한 기산: 입사일 기준 vs 회계연도 기준(회사 선택 옵션?).
2. 1일=소정근로시간 환산이 맞는지(반차=0.5일=소정/2), 시간연차 한도(JP 5일) 적용 여부.
3. 승인 단계: 1단계(인사 또는 총 승인)로 충분한지, 대리 신청 필요 여부.

### B-5. 확정 결정 (Phase 6-2 구현 — 사용자 "바로 진행" 지시로 아래 기본값 채택, 리뷰에서 조정 가능)

> 핵심 방침이 **법정 자동 계산 + 수동 조정**이므로, 자동값이 회사 실무와 달라도 **수동 부여(MANUAL grant)로 항상 보정 가능** → 아래 기본값은 되돌릴 수 없는 결정이 아니다.

- **D1. 기산 = 입사일(hire_date)**. `users.hire_date DATE` 신설(V16, 기존 행은 `DATE(created_at)` 백필). 회계연도 일괄부여 회사는 수동 부여로 대응(자동은 입사일 기준만).
- **D2. 연차 자동 계산 = 소재국 순수 함수 `LeaveAccrualPolicy`**(BreakPolicy와 동형 — DB/세션 무의존, 단위 테스트).
  - KR: 입사 1년 미만 = 경과 개월수(최대 11) / 1년↑ 15 / 3년↑ 2년마다 +1(최대 25). 출근율 80% 요건은 **충족 가정**(실제 출근율 미추적 — 문서화).
  - JP: 0.5년 미만 0 / 0.5년 10 / 근속표 1.5→11·2.5→12·3.5→14·4.5→16·5.5→18·6.5년↑ 20.
  - 파트타이머 비례부여(比例付与)는 1차 범위 외(주5일 정규 가정) — 후속.
- **D3. 법정 계산은 "제안" — 자동 부여 아님(사용자 피드백 반영)**. 법정 계산값은 관리자 화면에 **제안(미리보기)**으로만 표시(`suggestedAnnualMinutes`, 부여하지 않음). 관리자가 회사별로 확인 후 **"제안 적용"**을 눌러야 AUTO 연차 grant가 upsert되어 실제 잔여에 반영된다. 멤버 잔여 개요에 "법정 제안 vs 현재 잔여"를 나란히 보여 판단을 돕는다. cron/상시 적립 없음.
- **D4. 1일 = 개인 소정근로시간(분)** = (defaultWorkEnd − defaultWorkStart) − `BreakPolicy.requiredBreak`(그 구간). 예 09–18 KR = 540−60 = **480분**. 반차 = 소정/2. HOUR 종류는 실분 차감. 내부 저장은 전부 **분(minutes)**.
- **D5. 승인 1단계** — HR_ADMIN 또는 TENANT_ADMIN이 승인/반려. 멤버는 **본인 신청만**(대리 신청 후속). 승인 시에만 잔여 차감(취소·반려는 차감 없음/복원).
- **D6. 소멸** — grant.expires_on: KR 입사기념일+1년, JP +2년. 잔여 계산은 `asOf ≤ expires_on`인 grant만 합산.
- **D7. leave_type** — 테넌트별 커스텀 CRUD(인사·총관리자). **ANNUAL(연차)**는 내장 시드(is_annual, 자동계산 대상). 그 외(경조/병가/여름 등)는 회사 정의(paid·unit·requires_approval 지정).
- **경계**: 멤버 휴가 = `/api/v1/attendance/leave/**`(멤버+인사+총), 관리자 휴가 = `/api/v1/tenant/leave/**`(인사+총) — 기존 RoleInterceptor 규칙에 편승(신규 규칙 불요). 화면 W015(멤버 휴가) / W016(관리자 휴가 관리).
