## D28. 수동 정정 스탬프 — 사유 필수, 자동 스탬프와 데이터로 구분

- **결정**: `POST /api/v1/attendance/manual`(본인 전용) — 날짜·시각·타입(출근/퇴근/조퇴)·사유로 소급 등록.
  `attendance.source`(AUTO/MANUAL — V12)로 등록 경로를 구분하고, 사유는 코드 4종
  (FORGOT 찍는 것을 잊음 / DEVICE 단말·통신 / OFFSITE 외근·출장 / OTHER 기타) + 자유 텍스트(OTHER는 필수).
- **검증 규칙**: 미래 불가, 오늘−90일 이내, BREAK 제외(시작/종료 페어링 정합성 — 후속 후보).
  **상태머신 검사 없음** — 정정 목적이고, append-only라 원래 값도 행으로 남는다. 채용은 기존 규칙(마지막 값 우선) 그대로.
- **비고**: 좌표·장소 없음(terminal='manual') — 위치는 자동 스탬프의 무결성 장치이지 정정의 입력이 아님.

## D29. 조작 이력 노출 — 새 테이블이 아니라 "이미 있는" 행의 노출

- **발견**: `attendance`는 원래 **append-only**(출근 2번 = 2행) — 부족한 건 저장이 아니라 노출이었다.
- **결정**: `GET /api/v1/attendance/daily?date=`로 그 날짜의 전 스탬프(중복·수동·사유 포함)를 반환.
  W006에서 **행 클릭 → 일자 상세 모달**(테이블이 좁다는 제약의 해법 — 사유는 모달에서),
  테이블에는 날짜 셀의 "수동" 마커만(`DailyAttendance.manual`).
- **계약 명시**: 거부된 체크(allowed=false)는 확정 전 단계라 기록·노출 대상이 아니다.

## D30. 주말·공휴일 근무 — 멤버별 근무 요일 + 휴일 스탬프 집계 채용

- **결정**: `users.work_days` CHAR(7)(월~일, 기본 1111100 — V12). TA가 W009 스케줄 모달의 요일 칩으로 설정
  (`PUT /members/{id}/schedule`에 workDays 추가, 전 요일 휴무는 400).
- **집계(어셈블러)**: 공휴일·개인휴일·요일 휴무(dayOff)에도 **스탬프를 채용**(기존은 공란 스킵 — 휴일 근무가 조회에서 증발했다).
  스케줄이 없는 날의 법정휴게는 **실체류 기반** `policy.requiredBreak(체류)`, 총근무 공식은 동일 — 월 합계 포함.
  판정: 공휴일 > 일자 오버라이드(등록되면 근무일) > 요일 플래그.
- **스탬프 등록 자체는 원래 날짜 무관**(상태머신은 직전 기록만 본다) — "공휴일에도 찍을 수 있으면 된다"는 이미 충족, 변경 없음.
- **W005 오늘 스케줄**: 요일 휴무도 공휴일처럼 비표시.

## Phase 5.1~5.2 — 정정 UX 개편(피드백 반영)

- **D31. 모달 포커스 강탈 버그 수정(근본 원인)**: 공통 Modal이 `onClose`(렌더마다 새 인라인 함수)를 이펙트
  의존성으로 갖고 있어, W005 시계의 매초 리렌더마다 이펙트가 재실행되어 `panel.focus()`가
  열려 있는 select를 즉시 닫고 입력 중 포커스를 뺏었다("사유 옵션이 바로 닫힘", "상세 입력이 안 됨"의 정체).
  → onClose는 ref로 유지, 이펙트는 마운트 1회.
- **D32. 전용 폼 컴포넌트**: 네이티브 select/time을 `SelectField`(리스트 패널·바깥클릭/ESC 닫기·선택 항목
  센터 스크롤)와 `TimeField`(**"09:30" 단일 필드** — 누르면 시·분 2열이 한 패널로, 분 선택 시 닫힘)로 대체.
  출근/퇴근 행은 하나의 그룹 박스에 체크박스+시각으로 정렬(부품 나열 금지).
- **D33. 정정 모달 재설계**: 날짜는 클릭한 날로 고정(제목 표시, 입력란 없음) / **출근·퇴근 체크로 동시 등록**
  (둘 다 잊은 날 한 번에 — 등록은 출근→퇴근 순 2회 호출) / 사유 한 컬럼(직접 입력 선택 시에만 입력란).
- **D34. 오조작 방지**: 행 전체 클릭 폐지 — 날짜 셀의 밑줄 버튼만 일자 상세 진입점.
- **D35. 잘못 입력 복구 = 수정(삭제 금지)**: `PUT /attendance/manual/{id}` — 본인 + MANUAL 행의
  시각/구분/사유 변경(자동 기록 불변, 조건 불일치 404 존재 비노출). **이력 삭제는 제공하지 않는다**(운영 결정).
  일자 상세의 MANUAL 행 [수정] → 정정 모달이 수정 모드(값 프리필)로 열린다.
- **D36. 조퇴 폐지(UI)**: 출결 버튼과 정정 모달에서 조퇴 제거 — 출근/퇴근/휴식만.
  타입 자체(코드 3)는 과거 데이터 표기·집계 호환을 위해 유지(이력에 "조퇴"로 표시, 집계는 퇴근과 동일).
  참고: v1.0(5년 전)부터 조퇴는 별도 버튼이었음(시각 파생 아님) — 이번에 제품 결정으로 정리.
- 검증: `mvn test` 296건 그린, E2E 10체크 그린 — 통합 타임피커(시 선택 후 유지·분 선택 시 닫힘),

## Phase 5.3 — 휴식 시간 정정 + 인정 휴게 표시

- **D37. 휴식 시간 수동 정정(시작·종료 쌍)**: `POST /attendance/manual/break` { date, startTime, endTime, reason }
  → 휴식 시작(STATUS_ACTIVE)·종료(STATUS_BREAK_ENDED)를 한 번에 등록(단일 스탬프 정합성 회피).
  Phase 5에서 BREAK를 정정 대상에서 뺐던 제약을 해제. 정정 모달에 "휴식" 행(시작~종료) 추가,
  기존 휴식 스탬프는 [수정]으로 시각 정정(휴식↔근무 구분 전환은 400 — 시작/종료 status 보존).
- **D38. 인정 휴게 = max(법정, 실휴식)**: 월별 표의 `실휴식`+`법정휴게` 두 열을 **`휴게` 한 열**로 통합.
  휴식 미기록이어도 근무일이면 법정휴게로 자동 인정, 실휴식이 법정 초과면 실측 인정(총근무에서 차감).
  예: 8시간 근무 + 수동 45분 → 인정 60분 / 수동 90분 → 인정 90분. `DailyAttendance.recognizedBreakMinutes`
  신설(어셈블러 산출, 총근무 = 체류 − 인정 휴게). 인정>법정인 날은 강조(총계 감소 사유 시인성).
- 검증: `mvn test` 299건 그린(휴식 정정·수정·인정 휴게 경계 신규), E2E 8체크 그린
  (휴식 시작~종료 등록 → 월별 인정 1:30·근무 7:30·강조 → 상세에 시작/종료 표시 → 종료 13:30→13:00
  수정 → 인정 1:00·근무 8:00 재산출).
