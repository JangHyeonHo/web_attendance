## D28. 수동 정정 스탬프 — 사유 필수, 자동 스탬프와 데이터로 구분

- **결정**: `POST /api/v1/attendance/manual`(본인 전용) — 날짜·시각·타입(출근/퇴근/조퇴)·사유로 소급 등록.
  `attendance.source`(AUTO/MANUAL — V12)로 등록 경로를 구분하고, 사유는 코드 4종
  (FORGOT 찍는 것을 잊음 / DEVICE 단말·통신 / OFFSITE 외근·출장 / OTHER 기타) + 자유 텍스트(OTHER는 필수).
- **검증 규칙**: 미래 불가, 오늘−90일 이내, BREAK 제외(시작/종료 페어링 정합성 — 후속 후보).
  **상태머신 검사 없음** — 정정 목적이고, append-only라 원래 값도 행으로 남는다. 채용은 기존 규칙(마지막 값 우선) 그대로.
- **비고**: 좌표·장소 없음(terminal='manual') — 위치는 자동 스탬프의 무결성 장치이지 정정의 입력이 아님.

## D29. 조작 이력 노출 — 새 테이블이 아니라 "이미 있는" 행의 노출

- **발견**: `attendance`는 원래 **append-only**(출근 2번 = 2행) — 부족한 건 저장이 아니라 노출이었다.
- **결정**: `GET /api/v1/attendance/daily?date=`로 그 날짜의 전 스탬프(중복·수동·사유 포함)를 반환.
  W006에서 **행 클릭 → 일자 상세 모달**(테이블이 좁다는 제약의 해법 — 사유는 모달에서),
  테이블에는 날짜 셀의 "수동" 마커만(`DailyAttendance.manual`).
- **계약 명시**: 거부된 체크(allowed=false)는 확정 전 단계라 기록·노출 대상이 아니다.

## D30. 주말·공휴일 근무 — 멤버별 근무 요일 + 휴일 스탬프 집계 채용

- **결정**: `users.work_days` CHAR(7)(월~일, 기본 1111100 — V12). TA가 W009 스케줄 모달의 요일 칩으로 설정
  (`PUT /members/{id}/schedule`에 workDays 추가, 전 요일 휴무는 400).
- **집계(어셈블러)**: 공휴일·개인휴일·요일 휴무(dayOff)에도 **스탬프를 채용**(기존은 공란 스킵 — 휴일 근무가 조회에서 증발했다).
  스케줄이 없는 날의 법정휴게는 **실체류 기반** `policy.requiredBreak(체류)`, 총근무 공식은 동일 — 월 합계 포함.
  판정: 공휴일 > 일자 오버라이드(등록되면 근무일) > 요일 플래그.
- **스탬프 등록 자체는 원래 날짜 무관**(상태머신은 직전 기록만 본다) — "공휴일에도 찍을 수 있으면 된다"는 이미 충족, 변경 없음.
- **W005 오늘 스케줄**: 요일 휴무도 공휴일처럼 비표시.

## Phase 5.1~5.2 — 정정 UX 개편(피드백 반영)

- **D31. 모달 포커스 강탈 버그 수정(근본 원인)**: 공통 Modal이 `onClose`(렌더마다 새 인라인 함수)를 이펙트

  → onClose는 ref로 유지, 이펙트는 마운트 1회.
- **D32. 전용 폼 컴포넌트**: 네이티브 select/time을 `SelectField`(리스트 패널·바깥클릭/ESC 닫기·선택 항목
  센터 스크롤)와 `TimeField`(**"09:30" 단일 필드** — 누르면 시·분 2열이 한 패널로, 분 선택 시 닫힘)로 대체.
  출근/퇴근 행은 하나의 그룹 박스에 체크박스+시각으로 정렬(부품 나열 금지).
- **D33. 정정 모달 재설계**: 날짜는 클릭한 날로 고정(제목 표시, 입력란 없음) / **출근·퇴근 체크로 동시 등록**

- **D35. 잘못 입력 복구 = 수정(삭제 금지)**: `PUT /attendance/manual/{id}` — 본인 + MANUAL 행의

  일자 상세의 MANUAL 행 [수정] → 정정 모달이 수정 모드(값 프리필)로 열린다.
- **D36. 조퇴 폐지(UI)**: 출결 버튼과 정정 모달에서 조퇴 제거 — 출근/퇴근/휴식만.

  참고: v1.0(5년 전)부터 조퇴는 별도 버튼이었음(시각 파생 아님) — 이번에 제품 결정으로 정리.

## Phase 5.3 — 휴식 시간 정정 + 인정 휴게 표시

- **D37. 휴식 시간 수동 정정(시작·종료 쌍)**: `POST /attendance/manual/break` { date, startTime, endTime, reason }
  → 휴식 시작(STATUS_ACTIVE)·종료(STATUS_BREAK_ENDED)를 한 번에 등록(단일 스탬프 정합성 회피).
  Phase 5에서 BREAK를 정정 대상에서 뺐던 제약을 해제. 정정 모달에 "휴식" 행(시작~종료) 추가,
  기존 휴식 스탬프는 [수정]으로 시각 정정(휴식↔근무 구분 전환은 400 — 시작/종료 status 보존).
- **D38. 인정 휴게 = max(법정, 실휴식)**: 월별 표의 `실휴식`+`법정휴게` 두 열을 **`휴게` 한 열**로 통합.
  휴식 미기록이어도 근무일이면 법정휴게로 자동 인정, 실휴식이 법정 초과면 실측 인정(총근무에서 차감).
  예: 8시간 근무 + 수동 45분 → 인정 60분 / 수동 90분 → 인정 90분. `DailyAttendance.recognizedBreakMinutes`
  신설(어셈블러 산출, 총근무 = 체류 − 인정 휴게). 인정>법정인 날은 강조(총계 감소 사유 시인성).
- 검증: `mvn test` 299건 그린(휴식 정정·수정·인정 휴게 경계 신규), E2E 8체크 그린
  (휴식 시작~종료 등록 → 월별 인정 1:30·근무 7:30·강조 → 상세에 시작/종료 표시 → 종료 13:30→13:00
  수정 → 인정 1:00·근무 8:00 재산출).

## Phase 6-1 — 관리자 세분화(총관리자 / 인사관리자)

- **D39. 관리자 2단계 분리**: 회사 권한이 한 관리자에 집중되는 위험을 낮추기 위해 기존 `TENANT_ADMIN`을
  **총관리자(TENANT_ADMIN)** 와 **인사관리자(HR_ADMIN)** 로 나눔(멤버 제외 관리자 등급은 2단계 — 3단계 아님).
  권한 경계: 인사관리자 = 멤버·공휴일·(향후)휴가 관리 / 총관리자 = 위 전부 + **역할 지정**(관리자 임명) + **회사 메일 템플릿**.
  근거: 관리자 임명권과 대외 발송(메일) 설정을 총관리자로 격리해 직권 분산.
- **D40. 인가는 화이트리스트 유지(서열 도입 안 함)**: `RoleInterceptor` 규칙에 first-match로
  `/tenant/members/*/role`·`/tenant/mail-templates/**` → 총관리자 전용을 `/tenant/**` → {총,인사} 규칙보다 **앞에** 배치.
  네비게이션(`Screen` 허용 role Set)·프론트 헤더 메뉴·역할 열 SelectField 노출도 동일 경계로 3중 방어.
  마지막 활성 총관리자 보호(강등/비활성 409)를 **인사관리자 강등**까지 확장(`updateRole` 가드).
- **D41. `ck_users_role` CHECK 제약 갱신(라이브 발견)**: 단위 테스트는 매퍼를 목킹해 통과했으나,
  실 DB에는 V4의 `CHECK (role IN ('SYSTEM_ADMIN','TENANT_ADMIN','MEMBER'))`가 남아 있어
  HR_ADMIN 지정이 500(제약 위반)으로 실패 — **마이그레이션 리허설에서만 잡히는 결함**. V15에서
  `DROP CONSTRAINT ... ADD CONSTRAINT`로 HR_ADMIN 포함하도록 교체(+ role 컬럼 주석 정합).
  교훈: role/status처럼 enum을 DB CHECK로 강제하는 컬럼은 enum 변경 시 반드시 동반 마이그레이션 필요.
- 검증: `mvn test` **311건 그린**(HR_ADMIN 인가·네비·마지막관리자 가드 신규), 프론트 빌드 그린,
  실 DB **라이브 인가 스모크 8체크 그린** — 총관리자가 HR_ADMIN 지정(200) → 인사관리자 로그인·홈=출결(W005)
  → 멤버/공휴일 200 → 역할변경·메일템플릿 403(차단) → 총관리자 메일템플릿 200. V1→V15 라벨/제약 실적용.

## Phase 6-2 — 휴가(leave) 관리

설계·결정: `docs/plan/admin-roles-and-leave.md` §B(초안·검증요청) + §B-5(확정 결정). 방침 = **법정 자동 계산 + 수동 조정 / 일 단위 + 반일·시간**.

- **D42. 데이터 모델(V16)**: `leave_type`(테넌트별 종류 — ANNUAL 내장 시드) / `leave_grant`(부여, 분 단위, AUTO 연차는 유저·종류·연도 upsert / MANUAL 조정) / `leave_request`(신청·결재). `users.hire_date` 신설(연차 기산, 기존 행 created_at 백필). **잔여 = Σ유효부여 − Σ승인신청**, 신청 시 대기분까지 가용에서 차감해 초과 예약 방지.
- **D43. 법정 연차 자동 계산 = 순수 함수 `LeaveAccrualPolicy`(KR/JP)**: 소재국 기준 입사일→기준일 근속으로 부여 일수 산정(KR 1년미만 월1일·최대11 / 1년 15 / 3년~ 2년당+1·최대25, JP 0.5년 10 → 6.5년~ 20). BreakPolicy와 동형(DB/세션 무의존). 출근율 요건은 충족 가정. **법정 계산은 "제안"이며 자동 부여가 아니다**(사용자 피드백): 계산값은 관리자 화면에 제안(`suggestedAnnualMinutes` 미리보기)으로만 보이고, 관리자가 회사별로 확인 후 "제안 적용"을 눌러야 AUTO grant가 upsert되어 실제 잔여에 반영된다(cron 없음).
- **D44. 1일 = 개인 소정근로시간** = (근무구간 − 법정휴게, BreakPolicy 재사용). 반차=1/2, 시간휴가=실분. **근무일 계산**은 개인 근무 요일 + 공휴일 제외(토·일·공휴일 신청은 400).
- **D45. 승인 1단계**(인사/총관리자) — 멤버 본인 신청·취소(시작 전 PENDING/APPROVED). 승인만 차감, 반려·취소는 비차감/복원(상태 전이로 자동). 승인 종류 아닌 타입은 즉시 확정.
- **D46. 경계**: 멤버 `/attendance/leave/**`(전원) / 관리자 `/tenant/leave/**`(인사+총) — 기존 RoleInterceptor 규칙 편승(신규 규칙 불요). 화면 W015(멤버 휴가)·W016(관리자 휴가 관리, 탭: 결재/멤버 잔여/종류). 공유 표시 키는 공통 W999에 시드(양 화면 해석).
- **라이브에서만 잡힌 결함(수정)**: ① `@Select` 텍스트블록+문자열 결합이 경계 공백을 제거(`SELECT`+`r.`→`SELECTr`) → 뷰 쿼리를 일반 문자열 결합으로 교체. ② 요청 DTO의 원시 `boolean halfDay`가 필드 생략 시 역직렬화 500 → `Boolean`(null 허용, null=반차 아님). ③ i18n 공유 키 미해석(W016에서 W015 전용 키 참조) → W999로 이관.

### Phase 6-2 코드리뷰 반영 (전체 레벨 리뷰 9건)

- **D47. 동시 결재 초과 부여 방지**: 신청/승인 잔여 검사~기록을 `leave_grant` 행 `FOR UPDATE`(`lockByUserType`)로 직렬화 — 마지막 관리자 보호와 같은 규율. 두 관리자가 같은 멤버의 대기 2건을 동시 승인해도 잔여 초과 부여 불가.
- **D48. 잔여 표시 분 단위 정확화**: 0.5일 반올림 폐지. `formatLeaveAmount`(공용 util)로 '일 + 시간 + 분' 정확 표기(예 "15일 6시간 30분", 시간휴가 "2시간 30분"). W999 `UNIT_MIN` 라벨(V17).
- **D49. 본인 취소는 대기(PENDING)만**(시간 제약 폐지 — 당일 실수도 복구). 승인건은 관리자만. 프론트 취소 버튼도 대기에만 노출(백엔드 정책 일치).
- **D50. 기간 겹침 금지**: 같은 유저 PENDING/APPROVED와 겹치는 신청 400(`existsOverlap`, 이중 차감 방지).
- **D51. 비활성 종류 잔여 은폐 방지**: 잔여 조회를 활성 종류만이 아니라 부여·사용·대기가 있는 비활성 종류까지 노출.
- **D52. 월말 입사 연차 과소 계산 수정**: `completedMonths`(기념일 월말 클램프)로 1/31 입사→2/28을 1개월로 인정(KR/JP 공통).
- **D53. 연차(내장) 종류 단위·활성 잠금**: `updateType`에서 is_annual은 unit(DAY)·active 변경 차단(자동계산·조회 모순 방지).
- **D54. 정리**: 휴가 컨트롤러 `@Operation` summary를 메시지 키(`api.leave.*`)로 통일(Swagger 3개국어), 수량 포매터 중복(2화면) 공용 util로 추출.
