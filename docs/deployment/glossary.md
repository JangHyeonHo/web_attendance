# 용어집 (구축 문서용)

구축 문서에서 처음 나오는 순서대로. "정확한 정의"보다 "이 문서를 따라가는 데 필요한 만큼"만 설명한다.

## 계정·AWS

| 용어 | 뜻 |
|---|---|
| **루트 계정(root user)** | 가입 이메일로 로그인하는 전능 계정. 평소엔 쓰지 않고 봉인한다([02](02-aws-iam.md)) |
| **IAM** | AWS의 사용자·권한 관리 서비스. 일상 작업은 IAM 사용자로 한다 |
| **IAM 사용자** | 계정 안에 만드는 로그인 주체. 우리는 AdministratorAccess 정책을 붙인 관리자 1명을 만든다 |
| **정책(Policy)** | "무엇을 할 수 있는가"의 권한 목록. 사용자에게 붙여서 권한을 준다 |
| **MFA** | 비밀번호에 더해 휴대폰 인증 앱의 6자리 코드를 요구하는 2단계 인증 |
| **리전(Region)** | 데이터센터가 있는 지역(도쿄 ap-northeast-1, 서울 ap-northeast-2…). 콘솔 우상단에서 선택하며, 만든 자원은 그 리전 안에서만 보인다 |
| **EC2** | AWS의 가상 서버 서비스. "서버를 만든다" = "EC2 인스턴스를 시작한다" |
| **인스턴스(Instance)** | 가상 서버 1대 |
| **인스턴스 유형** | 서버의 하드웨어 구성(CPU 개수·메모리). t4g = ARM(Graviton) 계열 저가형 |
| **AMI** | 인스턴스의 초기 디스크 이미지(OS). 우리는 Ubuntu Server 24.04 (Arm) |
| **키 페어(Key Pair)** | 서버 접속용 열쇠 파일(.pem). 생성 때 한 번만 다운로드 가능 — 분실 주의 |
| **보안 그룹(Security Group)** | 클라우드 쪽 방화벽(어느 포트를 열지). AWS에서는 사실상 유일한 방화벽이다 |
| **인바운드(Inbound)** | 밖→서버 방향 통신 규칙. 반대말은 아웃바운드(서버→밖) |
| **Elastic IP** | 인스턴스를 중지/재시작해도 유지되는 고정 공인 IP |
| **EBS / gp3** | 인스턴스에 붙는 디스크(볼륨)와 그 표준 등급 이름 |
| **예산(Budgets)** | 월 요금이 설정액에 근접하면 메일로 알려주는 기능 — 과금 사고 방지용 |
| **VPC** | 내 자원들이 소속되는 가상 네트워크. 우리는 전용 `mt-dev-vpc`를 설계해 쓴다([03](03-aws-network.md)) |
| **서브넷(Subnet)** | VPC 안의 구획. 퍼블릭(인터넷 노출)/프라이빗(내부 전용)으로 나눈다 |
| **CIDR** | 네트워크 주소 범위 표기(예: 10.0.0.0/16). 숫자가 클수록 좁은 범위 |
| **인터넷 게이트웨이(IGW)** | VPC와 인터넷 사이의 통로. 퍼블릭 서브넷의 필수품 |
| **NAT 게이트웨이** | 프라이빗 서브넷이 밖으로 나갈 때 쓰는 유료 장비(월 $30대) — ★우리 구성에선 만들지 않는다★ |

## 접속·서버 안

| 용어 | 뜻 |
|---|---|
| **SSH** | 원격 서버에 터미널로 접속하는 표준 방법. `ssh` 명령 |
| **SSH 키(공개키/개인키)** | 비밀번호 대신 쓰는 파일 열쇠 한 쌍. 공개키(.pub)는 서버에 등록, 개인키는 내 PC에 보관(유출 금지) |
| **iptables** | 리눅스 OS 안의 방화벽. AWS 우분투는 기본 비활성이라 우리 구성에서는 보안 그룹만 신경 쓰면 된다 |
| **systemd / 서비스** | 리눅스의 프로그램 상주 관리자. "서비스로 등록" = 부팅 시 자동 시작 + 죽으면 재시작 |
| **nano** | 터미널용 간단 텍스트 편집기. `sudo nano <파일경로>`로 열고, Ctrl+O(저장)·Enter·Ctrl+X(닫기) |
| **환경변수** | 프로그램에 설정값을 주입하는 통로. 우리는 `/etc/attendance/attendance.env` 파일에 모아 둔다 |

## 웹·도메인

| 용어 | 뜻 |
|---|---|
| **DNS** | 도메인 이름 → IP 주소 변환 체계 |
| **A 레코드** | "이 이름은 이 IP다"라는 DNS 등록 항목. `app.<도메인> → 서버 IP` 로 1건 등록한다 |
| **와일드카드 레코드/인증서** | `*.app.<도메인>`처럼 하위 이름 전부를 한 번에 커버하는 것. 테넌트 서브도메인(12번 문서)에서 필요 |
| **TLS/HTTPS 인증서** | 브라우저 자물쇠. Let's Encrypt에서 무료 발급(certbot 도구) |
| **리버스 프록시** | 브라우저의 요청을 받아 뒤의 백엔드로 전달하는 앞단 서버 — 우리 구성의 nginx 역할 |
| **레지스트라** | 도메인을 구매·관리하는 업체(Cloudflare, 가비아, お名前.com 등) |

## 앱 쪽

| 용어 | 뜻 |
|---|---|
| **프로파일(dev/prod)** | Spring 설정 묶음 전환 스위치. 공개 서버는 항상 `prod`([01](01-architecture.md)) |
| **Flyway / 마이그레이션** | DB 스키마를 버전 파일(V1, V2, …)로 관리하는 도구. 앱이 기동하며 자동 적용 — 수동 DDL 불요 |
| **jar** | 자바 앱을 파일 하나로 묶은 실행 파일. `java -jar app.jar`로 실행 |
| **테넌시(우리 서비스의)** | 우리 SaaS의 "고객사" 단위. 클라우드 용어의 테넌시와는 무관한 동음이의어이니 혼동 주의 |
