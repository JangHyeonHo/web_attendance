# 03. 네트워크(VPC) 설계와 생성

> 이전: [02. AWS 계정 준비](02-aws-iam.md) · 다음: [04. EC2 인스턴스 만들기](04-aws-ec2.md) / [인덱스](README.md)

인스턴스보다 네트워크가 먼저다. 기본(default) VPC에 얹지 않고 **이 서비스 전용 VPC를
이름 규칙과 함께** 만든다 — 격리(다른 실험·리소스와 안 섞임), 추적(이름만 봐도 용도를 앎),
확장(나중에 매니지드 DB용 프라이빗 서브넷을 같은 설계 안에 추가) 때문이다.
VPC·서브넷·라우팅은 전부 무료라 제대로 만들어도 비용 차이가 없다.

## 3-1. 설계 (명칭 규칙 포함)

명칭 규칙: `mt-<환경>-<자원>[-<상세>]` — 환경은 dev(테스트)/prod(운영 예약).

| 자원 | 이름 | 값 | 용도 |
|---|---|---|---|
| VPC | `mt-dev-vpc` | 10.0.0.0/16 | 이 서비스 전용 네트워크 |
| 퍼블릭 서브넷 | `mt-dev-subnet-public1-<AZ>` | 10.0.0.0/24 | 웹 서버(EC2)가 사는 곳 |
| 인터넷 게이트웨이 | `mt-dev-igw` | — | VPC↔인터넷 통로 |
| 라우트 테이블 | `mt-dev-rtb-public` | 0.0.0.0/0 → igw | 퍼블릭 서브넷의 길안내 |
| 보안 그룹 | `mt-dev-sg-web` | 인바운드 22/80/443 | 웹 서버 방화벽(아래 3-3) |
| (예약) 프라이빗 서브넷 | `mt-dev-subnet-private1-<AZ>` | 10.0.128.0/24 | 운영 전환 시 매니지드 DB 자리 — 지금은 비워 둠 |

- CIDR(10.0.0.0/16)은 사내망·VPN(예: WireGuard의 10.x 대역)과 겹치지 않는지만 확인.
  겹치면 172.31 대신 10.10.0.0/16처럼 비켜 잡는다
- prod는 이 VPC를 공유하지 않는다 — 운영 전환 시 `mt-prod-vpc`를 따로 만든다
  (테스트 실험이 운영 네트워크에 닿을 수 없게)

## 3-2. VPC 생성 (마법사 한 화면)

1. 콘솔 검색창 "VPC" → **VPC 생성(Create VPC)**
2. **"VPC 등(VPC and more)"** 선택 — VPC·서브넷·IGW·라우트 테이블을 위 설계대로
   한 번에 만들어 주고, 이름도 접두사로 일괄 부여된다:
   - 이름 태그 자동 생성: **`mt-dev`** 입력 (→ `mt-dev-vpc`, `mt-dev-subnet-…` 자동 명명)
   - IPv4 CIDR: `10.0.0.0/16`
   - 가용 영역(AZ) 수: **1** / 퍼블릭 서브넷 수: **1** / **프라이빗 서브넷 수: 1**
     (프라이빗은 무료이고, 운영 전환 때 DB 자리로 쓸 예약 — 지금은 아무것도 안 둔다)
   - **NAT 게이트웨이: 없음**(★유료 장비 — 지금 구성에 불필요, 켜면 월 $30대가 나간다★)
   - VPC 엔드포인트: 없음 / DNS 옵션: 둘 다 체크 유지
3. **VPC 생성** → 미리보기 다이어그램에서 위 표와 같은 구조인지 확인

## 3-3. 보안 그룹 생성 (이름 있는 방화벽)

인스턴스 마법사가 만들어 주는 `launch-wizard-1` 같은 익명 그룹을 쓰지 않고 직접 만든다:

1. VPC 콘솔 왼쪽 **보안 그룹** → **보안 그룹 생성**
2. 이름 `mt-dev-sg-web` / 설명 `web server (ssh/http/https)` / VPC: **mt-dev-vpc** 선택
3. **인바운드 규칙** 3건:

| 유형 | 소스 | 비고 |
|---|---|---|
| SSH (22) | 내 IP | 전체 공개 금지 — IP 바뀌면 여기서 갱신 |
| HTTP (80) | 0.0.0.0/0 | 인증서 발급·https 리다이렉트 |
| HTTPS (443) | 0.0.0.0/0 | 서비스 본체 |

4. 아웃바운드는 기본(전체 허용) 유지 → 생성
5. **3306(DB)은 지금도, 앞으로도 추가하지 않는다** — DB는 서버 안에서만 접속

## 완료 확인

- [ ] VPC `mt-dev-vpc`(10.0.0.0/16)와 퍼블릭/프라이빗 서브넷이 이름 규칙대로 생성됨
- [ ] NAT 게이트웨이 **0개**(비용 확인 — 있으면 삭제)
- [ ] 보안 그룹 `mt-dev-sg-web` 인바운드가 22(내 IP)/80/443 세 개뿐
- [ ] 기본(default) VPC에는 아무것도 만들지 않았음
