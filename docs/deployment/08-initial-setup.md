# 08. 초기 설정과 첫 검증

> 이전: [07. DNS·TLS](07-dns-tls.md) · 다음(운영): [09. 배포 갱신](09-deploy-update.md) / [인덱스](README.md)

## 초기 계정 — ★가장 먼저 할 일★

Flyway 시드가 운영사(시스템) 관리자 계정을 만든다:

```
이메일: admin@attendance.local
비밀번호: Admin123!
```

**이 크리덴셜은 공개 저장소의 마이그레이션 파일에 적혀 있다.**
서버가 열리면 무엇보다 먼저 이 계정으로 로그인해 비밀번호를 변경한다.

## 첫 테넌트 개설 흐름

1. 운영사 관리자 로그인 → 테넌트 생성(회사 코드·이름·소재국·관리자 이메일)
2. 시스템이 테넌트 관리자에게 **초대 메일** 발송 → 메일의 링크에서 비밀번호 설정
   - 메일이 안 오면: SMTP 설정([05](05-backend-service.md)) 또는 발신 도메인의 스팸 분류 확인
   - 링크 도메인이 이상하면: `APP_MAIL_LINK_BASE_URL`이 실제 접속 URL과 다른 것
3. 테넌트 관리자 로그인 → 멤버 등록(멤버도 초대 메일로 비밀번호 설정)
4. 관리자 화면에서 공휴일 동기화 실행 — 외부 API(date.nager.at) 아웃바운드 확인을 겸한다

## 첫 검증 시나리오 (전부 통과해야 공개)

- [ ] `https://app.<도메인>` 접속 → 랜딩 표시, http → https 리다이렉트
- [ ] admin@attendance.local 로그인 → **비밀번호 변경 완료**
- [ ] `/swagger-ui.html` → 404 (prod 프로파일 확인)
- [ ] 테넌트 생성 → 초대 메일 실제 수신 → 비밀번호 설정 성공
- [ ] 멤버로 출근·퇴근 스탬프 → 출결 조회에 즉시 반영
- [ ] 위치 권한 허용/거부 각각에서 스탬프 등록 가능(거부 시 좌표 없이 등록되는 것이 정상)
- [ ] 공휴일 동기화 성공(해당 연도 공휴일 목록 표시)
- [ ] 언어 전환(한/영/일) 정상 — 키 이름이 그대로 보이는 라벨이 있으면 언어 마스터 누락
- [ ] `journalctl -u attendance`에 출결 파라미터 debug 로그가 없는지(프로파일 재확인)
- [ ] 백업 크론 첫 실행 결과 확인([10](10-backup-restore.md))
