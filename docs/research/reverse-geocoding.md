# 위치 → 장소명 변환(역지오코딩) — 무료 옵션 조사 (2026-07)

> 상태: **조사 완료·의사결정 대기**(도입 여부·시기 미정).
> 배경: v1부터 스탬프에 장소명 자리(`place_info`)가 있었으나 v1은 화면·DB 모두 "미완성" 문자열로
> 남아 있던 미구현 기능이고, v2는 좌표(위도·경도)만 기록하고 `place_info`는 null로 보낸다.
> 목적: 스탬프의 좌표를 사람이 읽는 주소/장소명으로 바꿔 **표시·증적 보조**에 쓰는 것.
> (사업장 반경 판정 = 지오펜싱은 외부 API가 필요 없는 별개 기능 — 이 문서 범위 밖)

## 결론 요약 — 무료로 한·일 커버 가능

**나라별로 무료 API를 분기해서 쓰면 비용 없이 커버된다**: 한국 = 카카오 로컬, 일본 = 국토지리원(GSI),
그 외 좌표 = 표시 생략(또는 OSM Nominatim). 예상 호출량이 스탬프 1건당 1콜(멤버 1인 하루 2~6콜)
수준이라 어느 무료 한도에도 한참 못 미친다.

| 옵션 | 무료 한도 | 커버리지 | 등록 | 비고 |
|---|---|---|---|---|
| **카카오 로컬 `coord2address`** | 일 10만 / 월 300만 | 한국만 | 카카오 디벨로퍼스 앱 키 | 한국 주소 품질 최상. 무료 쿼터 초과분은 유료 전환 신청제 |
| **국토지리원(GSI) 리버스 지오코더** | 무제한(공개 API) | 일본만 | 불요 | `mreversegeocoder.gsi.go.jp/reverse-geocoder/LonLatToAddress`. 정부 제공·키 없음. 町丁目 수준 주소. 실험적 성격이라 SLA 없음 — 실패 시 표시 생략으로 안전 처리 필요 |
| Google Geocoding API | 월 1만 건(2025-03부터 $200 크레딧 폐지 → SKU별 무료 한도) | 전 세계 | 구글 클라우드 결제수단 등록 | 초과 시 $5/1,000. **약관상 결과의 영구 저장·지도 외 사용에 제약이 있어**(캐시 기간 제한 등) DB에 장소명을 영구 보존하는 우리 용도와 충돌 소지 — 채택 시 약관 재검토 필수 |
| OSM Nominatim(공용) | 1 req/s 정책 | 전 세계 | 불요 | 대량·상용 이용 비권장(공용 서버 정책). 출처 표기(ODbL) 필요. 자체 호스팅하면 무제한이나 운영 부담 |

- v1에서 "무료분만 썼던" 기억은 당시 구글 $200 월 크레딧 체계(2025-02 종료)였을 가능성이 높다.
  현행 구글은 크레딧이 아니라 API별 무료 건수 체계로 바뀌었다.

## 권장 설계(도입 시)

1. **서버에서 등록 시 1회 변환** — confirm(확정) 시점에 좌표 → 주소를 조회해 `place_info`에 저장.
   스탬프는 append-only·불변이므로 "등록 순간의 장소명"으로 확정되는 것이 증적 성격과 맞는다.
   프론트에서 변환하지 않는 이유: API 키 노출 방지 + 해시 검증(payload 불변) 흐름 유지.
2. **나라 분기**: 좌표의 국가 판별은 테넌트 소재국(`tenant.country`) 기준이면 충분
   (KR → 카카오, JP → GSI). 소재국 밖 좌표는 그대로 표시 생략(null 유지).
3. **실패 무해화**: 변환 실패(타임아웃·한도)는 스탬프 등록을 막지 않고 `place_info` null로 진행.
   외부 API 응답 지연이 출퇴근 UX를 해치지 않도록 짧은 타임아웃(1~2초) + 비동기 사후 채움도 검토.
4. 일자 상세·근무 중 멤버 조회 등 좌표를 보여주는 곳에서 장소명을 함께 표시.
