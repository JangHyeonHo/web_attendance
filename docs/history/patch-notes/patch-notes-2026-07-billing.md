# 패치 노트 — 인당(seat) 과금 + 청구서

Phase 7. 회사(테넌트)별로 **활성 인원당 이용료**를 과금하고, 운영사(SYSTEM_ADMIN)가 회사별 단가를
설정하며, 회사 총관리자(TENANT_ADMIN)가 자사 **월별 청구서**를 조회한다.

## 리서치 배경 (요금 모델 결정)

- **인당 과금(seat 기반)은 한·일 근태 SaaS의 공통 표준**. 일본(KING OF TIME ¥330/인, ジョブカン ¥200~500/인)은
  인당 과금이 완전한 표준이고, 한국도 인당 과금을 쓴다(시프티 인당 3,000원, flex 인당 7,000원·VAT 별도).
- 한국 시장 실단가 밴드 ≈ **인당 3,000~7,000원(VAT 별도)**. 다수 벤더는 단가를 "견적 문의"로 비공개.
- 결정: **기본 단가 인당 2,000원**(침투가 — 시장 하한보다 낮게, 회사별 조정 가능하니 기본만 낮게),
  **VAT 별도**, **5인 이하 무료**(도입 전환율↑), 과금 기준 **월중 최대 활성 수**.

## D61. 데이터 모델 (V22)

- 기존 `tenant_billing`에 **`per_seat_amount`(기본 2000)·`free_seats`(기본 5)** 확장 — 운영사가 회사별로 설정
  (별도 화면 대신 기존 W008 결제정보 확장, 결제수단·청구이메일과 한 곳).
- **`tenant_seat_usage`(tenant_id, ym, max_seats)** — 월중 최대 활성 좌석 high-water mark.
  멤버 활성/비활성·초대 활성화와 청구서 조회 시 현재 활성 수를 세어 `GREATEST`로만 올린다(피크 보존).
  활성 수가 **증가하는 지점**(초대 비밀번호 설정 → ACTIVE, 상태 재활성)에 훅을 걸어 피크를 놓치지 않는다.
- **`invoice`(tenant_id, ym, ...)** — 마감(확정) 청구서 스냅샷. 행 존재 = 그 달 확정(ISSUED).
  단가·무료인원·최대인원을 확정 시점 값으로 스냅샷해 **이후 단가 변경이 과거 청구서에 소급되지 않는다**.

## D62. 과금 계산 (BillingService)

- 좌석 산정: **ACTIVE·비삭제·SYSTEM_ADMIN 제외**(운영사 계정은 회사 좌석이 아님).
- 과금식(전부 원, VAT 별도 라인):
  `과금인원 = max(0, 월중최대 − 무료)` · `공급가 = 과금인원 × 단가` · `부가세 = round(공급가×10%)` · `합계 = 공급가 + 부가세`.
- **진행 중인 달 = 잠정(PROVISIONAL, 실시간 계산)**, **마감 = 확정(ISSUED, 스냅샷)**.
  마감은 운영사가 실행하며 이미 확정된 달 재마감은 409(확정 청구서 불변). 청구월 형식 오류는 400.
- `touchSeatUsage`는 부가 기능 — 실패해도 본 요청을 깨지 않는다(경고 로그만, 감사 서비스와 동일 원칙).

## D63. API·인가

- 운영사(SYSTEM_ADMIN, `/system/**`): `PUT /system/tenants/{id}/billing`(단가·무료 인원 포함),
  `GET /system/tenants/{id}/invoices`, `POST /system/tenants/{id}/invoices/{ym}/close`.
- 회사 총관리자(TENANT_ADMIN 전용): `GET /tenant/billing/invoices`(자사, 세션 tenantId). **재무 정보라 인사관리자(HR_ADMIN) 제외** —
  RoleInterceptor에 `/api/v1/tenant/billing/**` → TENANT_ADMIN 규칙을 일반 `/tenant/**`보다 먼저 선언(첫 매칭).

## D64. 화면 (V23)

- **운영사 W008(테넌트 상세) 확장**: 결제정보에 인당 단가·무료 인원 입력, 월별 청구서 표 + PROVISIONAL 행 **마감** 버튼.
- **회사 W018(청구서) 신설**: 자사 월별 청구서(최근 12개월) — 청구월·최대/과금 인원·단가·공급가·부가세·합계·상태. TENANT_ADMIN 헤더 메뉴 추가.
- 공유 컬럼/상태 라벨은 W999(공통), 화면 전용은 각 윈도우. 3개 언어(한/영/일).

- 프론트 `tsc -b && vite build` 그린. V1→V23 마이그레이션 적용.
- 라이브 스모크: 인당 2,000·무료 1 설정 → 활성 2명 → **잠정 2,200원(과금 1명 + VAT 200)** → 마감 **확정 2,200원** →
  재마감 **409** / 잘못된 월 **400** / 회사 총관리자 자사 청구서 조회(최근 12개월) 확인.
- ⚠ 운영 주의: 실제 결제(PG) 실행은 이번 범위 밖(설정 + 청구서 표시까지). 통화 KRW.
  월중 최대는 활성 증가 이벤트·조회 시점에 기록 — 아무도 조회하지 않은 채 발생·소멸한 순간 피크는 이론상 누락 가능(마감/조회 시 확정).
